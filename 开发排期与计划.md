#### 开发任务清单

- [ ] 创建 `services/` 包结构
- [ ] 实现 `LLMAdapter` 统一调用入口，根据 `ModelStrategy.task_type` 查询对应 Provider
- [ ] 实现 `OpenAIProvider`（兼容 DeepSeek，因其 API 格式相同）
- [ ] 实现 `AnthropicProvider`（Claude 系列）
- [ ] 实现 `CustomProvider`（自定义 endpoint，兼容 OpenAI 格式）
- [ ] 实现 `CallTracker` 封装，自动记录每次调用的 token 数、耗时、状态到 `AICallLog`
- [ ] 改造 `POST /ai/providers/{id}/test` 端点，发送 "Hello" 到真实 LLM 验证连接
- [ ] 错误处理：超时重试（最多 2 次）、API Key 无效提示、额度不足提示

#### 技术要点

```
调用链路：LLMAdapter.call(task_type, prompt)
  → 查 ModelStrategy(task_type → provider_id)
  → 查 AIProvider(id, api_key, model, endpoint_url)
  → 路由到对应 ProviderAdapter
  → httpx 调用远程 API（timeout=120s）
  → CallTracker 记录日志
  → 返回 {content, input_tokens, output_tokens, model}
```
