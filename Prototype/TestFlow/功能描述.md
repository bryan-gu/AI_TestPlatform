# TestFlow 测试管理平台 — 按钮与操作功能描述

> 本文档描述每个页面上每个按钮/交互元素的预期功能行为，供前端开发参考。

---

## 一、全局交互

### 1.1 侧边栏 (Sidebar)

| 元素 | 位置 | 功能描述 |
|------|------|---------|
| **收起按钮** `<` | 侧边栏顶部右侧 | 收起侧边栏为图标模式，只显示图标和 tooltip，隐藏文字。状态持久化到 `localStorage`。 |
| **展开按钮** `>` | 侧边栏收起后显示 | 展开侧边栏恢复完整宽度，显示图标+文字。状态持久化到 `localStorage`。 |
| **导航项** | 侧边栏各分组 | 点击切换到对应页面，高亮当前选中项。支持分组：项目、知识、AI、系统管理。收起状态下 hover 显示 tooltip。 |
| **用户头像行** | 侧边栏底部 | 显示当前登录用户姓名+角色。点击可展开用户菜单（设置、退出登录等，当前为占位）。 |

### 1.2 顶栏 (Header)

| 元素 | 功能描述 |
|------|---------|
| **页面标题** `#page-title` | 根据当前页面动态显示标题（如"项目总览"、"测试用例"等），跟随 `switchPage` 切换。 |
| **搜索框** `header-search` | 点击打开全局搜索对话框（`dialog-search`），支持输入关键词搜索项目、用例、文档、用户。搜索结果展示匹配项，点击跳转对应页面。 |
| **主操作按钮** `#main-btn` | 根据当前页面动态变化：项目总览页隐藏；其他页面显示对应的新建/执行操作。按钮文字和颜色随页面变化。 |

### 1.3 全局组件

| 元素 | 功能描述 |
|------|---------|
| **Toast 通知** `#toast` | 右上角弹出提示，支持四种类型：`success`（绿色对勾）、`error`（红色叉号）、`info`（蓝色感叹号）、`warning`（黄色三角）。自动 3 秒消失。所有保存、删除、操作反馈统一使用此组件。 |
| **对话框遮罩** `.dialog-overlay` | 打开时显示半透明黑色遮罩，点击遮罩关闭对话框。按 `Esc` 键也可关闭。 |
| **删除确认对话框** `dialog-delete-confirm` | 通用删除确认弹窗。显示删除警告图标 + "确定要删除该条记录吗？" + "删除后数据将无法恢复"。点击"确认删除"后关闭弹窗 + toast 提示"已删除" + 当前行淡出移除。点击"取消"仅关闭弹窗。 |
| **全局搜索对话框** `dialog-search` | 输入框支持实时搜索，下方显示"最近访问"列表。搜索结果应匹配项目名、用例编号/标题、文档名、用户名。点击结果项关闭对话框并跳转到对应页面。 |

## 二、项目总览页 (`project.html`)

| 元素 | 功能描述 |
|------|---------|
| **统计卡片** (4张) | 展示进行中项目数、总测试用例数、通过率、待修复缺陷数。数据从后端获取。点击卡片无交互（纯展示）。 |
| **"全部查看"链接** | 项目列表卡片右上角。点击跳转到 `project-list`（项目管理页）。 |
| **项目列表表格行** | 展示最近 4 个项目。点击某行可跳转到该项目的知识库页（`knowledge`），查看该项目下的 Sprint 列表。 |
| **近期动态列表** | 时间线展示最近 5 条操作记录。纯展示，无交互。 |

## 三、项目管理页 (`project-list.html`)

| 元素 | 功能描述 |
|------|---------|
| **统计卡片** (4张) | 同项目总览，纯展示。 |
| **编辑按钮** (每行) | 点击打开"编辑项目"对话框（`dialog-project-edit`），表单预填充当前行的项目名称、描述、状态、进度、负责人。修改后点"保存"关闭对话框 + toast"已保存"。 |
| **删除按钮** (每行) | 点击打开通用删除确认对话框。确认后 toast"已删除" + 当前行 DOM 淡出移除。 |
| **新建项目** | 通过顶栏主按钮触发（当前页主按钮文字为"新建项目"）。打开"新建项目"对话框（`dialog-project-create`），填写项目名称、描述、状态、负责人、开始日期。点击"创建"关闭对话框 + toast"项目已创建" + 刷新列表。 |

## 四、测试用例页 (`testcase.html`)

| 元素 | 功能描述 |
|------|---------|
| **统计卡片** (4张) | 全部用例数、已通过数、未通过数、未执行数。纯展示。 |
| **项目筛选下拉** | 选择项目后过滤表格只显示该项目的用例。选项：全部项目、电商平台、支付系统。 |
| **批量执行按钮** | 点击后将表格中所有"待执行"状态的用例批量标记为"通过"，toast 提示"已批量执行 N 条用例"。 |
| **用例行** | 点击用例行（非操作区域）展开详情面板，显示前置条件、测试步骤、预期结果。再次点击收起。 |
| **编辑按钮** (每行) | 打开"编辑用例"对话框（`dialog-testcase-edit`），预填充用例编号（只读）、标题、优先级、执行状态、执行人。 |
| **删除按钮** (每行) | 打开通用删除确认对话框。确认后行淡出移除。 |
| **新建用例** | 主按钮文字为"新建用例"。打开"新建用例"对话框（`dialog-testcase-create`），填写所属项目、用例标题、优先级、执行人、前置条件、测试步骤、预期结果。 |

| **Sprint 编辑按钮** (每行) | 打开编辑 Sprint 对话框，可修改 Sprint 名称、描述。 |
| **sprint_all 行的"图谱"按钮** | 跳转到 `graph-list`，查看该项目的知识图谱汇总。 |

## 八、文档预览页 (`doc-preview.html`)

| 元素 | 功能描述 |
|------|---------|

## 九、知识图谱列表页 (`graph-list.html`)

| 元素 | 功能描述 |
|------|---------|
| **项目选择器** | 下拉切换当前项目。切换后过滤显示该项目的图谱。 |

| **"刷新"按钮** (需更新状态的图谱) | 触发重新生成该图谱，同"重新生成"按钮逻辑。 |

## 十、知识图谱详情页 (`graph.html`)

| 元素 | 功能描述 |
|------|---------|
| **面包屑导航** | 三级：项目名 → 图谱列表 → 当前图谱。 |
| **图谱信息卡片** | 显示图谱名称、描述、所属项目、创建人、更新时间。纯展示。 |
| **统计卡片** (4张) | 节点数、关联数、文档覆盖率、孤立节点数。纯展示。 |
| **图谱筛选标签** (全部/依赖/引用/覆盖) | 切换显示不同类型的关联边。`active` 标签高亮，切换后 SVG 图谱中只显示对应类型的边线，其他变灰或隐藏。 |
| **SVG 图谱画布** | 可视化展示节点和关联关系。节点可点击，点击后右侧面板更新为该节点的详情。 |
| **节点详情面板** | 显示选中节点的名称、ID、类型图标、关联文档数、强依赖数、上游依赖列表、下游影响列表。点击列表中的关联项可选中对应节点。 |
| **覆盖状态面板** | 展示需求覆盖率、测试用例覆盖、接口覆盖的进度条。纯展示。 |
| **最近变更面板** | 展示图谱最近的变更记录（新增需求、更新定义等）。纯展示。 |

## 十四、AI 工作台页 (`ai-workbench.html`)

| 元素 | 功能描述 |
|------|---------|
| **全量模式卡片** | 点击选中（高亮边框+浅蓝背景），表示从零执行 SKILL 全部 4 阶段。其他卡片取消选中。 |
| **增量模式卡片** | 点击选中，表示基于当前 Sprint 快照仅分析变更部分。其他卡片取消选中。 |
| **项目下拉选择器** | 选择要执行 SKILL 的项目。切换后下方 Sprint 列表刷新为该项目的 Sprint。 |
| **Sprint 标签组** | 点击选中某个 Sprint（高亮为蓝色实底），支持 Sprint 0、1、2、3 和 sprint_all。选中后 SKILL 流水线上下文切换到该 Sprint。 |
| **SKILL 流水线面板** | 展示 4 个阶段的执行状态：已完成（绿色对勾）、执行中（蓝色旋转动画+进度条）、等待中（灰色数字）。 |
| **阶段展开/收起** | 已完成的阶段可点击展开查看详细结果（分析结果摘要、Token 消耗等）。再次点击收起。 |
| **执行 SKILL 按钮** (主按钮) | 三态切换：① idle 状态显示"执行 SKILL"，点击开始执行；② 运行中显示"暂停执行"（橙色），点击暂停；③ 暂停后显示"继续执行"，点击恢复；④ 全部完成后显示"重新执行"。运行时显示计时器（实时更新已运行时间）。 |
| **知识图谱预处理面板** | 展示预处理状态（已完成/进行中）、节点数、关联边数。纯展示。"查看知识图谱 →" 链接跳转到 `graph-list`。 |
| **执行历史面板** | 展示最近 3 次执行记录，包含执行名称、时间、耗时、状态。纯展示。 |

| **添加服务商链接** | 打开"添加 AI 服务商"对话框（`dialog-ai-provider-add`），选择服务商类型（OpenAI/Anthropic/DeepSeek/自定义），填写 API Key、模型、端点 URL、温度、最大 Token。 |
| **服务商编辑按钮** (每行) | 打开"编辑 AI 服务商"对话框（`dialog-ai-provider-edit`），预填充当前行的服务商配置。 |
| **服务商测试按钮** (每行) | 点击后按钮显示加载动画（约 1.5 秒），然后 toast 显示测试结果："连接成功（响应时间 xxxms）"或"连接失败：xxx"。 |
| **服务商删除按钮** (每行) | 打开 AI 服务商删除确认对话框（`dialog-ai-provider-delete`），提示"删除后相关任务的模型分配将失效"。确认后行淡出移除。 |
| **模型分配策略 — 编辑链接** | 打开"编辑模型分配策略"对话框（`dialog-ai-strategy-edit`），为每种任务类型（需求文档分析、测试用例生成、自动化脚本生成、知识图谱关联、测试报告摘要）分配模型和温度参数。 |
| **全局参数 — 编辑链接** | 打开"编辑全局参数"对话框（`dialog-ai-global-edit`），修改最大 Token 数、超时时间、重试次数、请求并发数、日志级别。 |
| **调用日志筛选下拉** (2个) | 按服务商和状态筛选调用日志。选择后表格过滤显示匹配的日志条目。 |
| **调用日志表格** | 展示最近的 API 调用记录。纯展示。 |

| 测试范围 | 多行文本 | 描述纳入报告的用例范围 |

提示信息："报告将基于已执行的测试用例结果自动生成"。

| 字段 | 类型 | 说明 |

### 16.8 邀请用户 (`dialog-user-invite`)

| 字段 | 类型 | 校验 |
|------|------|------|
| 邮箱地址 | 邮箱输入 | 必填，格式校验 |
| 姓名 | 文本输入 | 必填 |
| 角色 | 下拉选择 | 项目管理员/测试工程师/只读观察员 |
| 所属项目 | 下拉选择 | 必选 |

提示信息："系统将发送邀请邮件至该邮箱，用户需通过邮件激活账号"。

### 16.9 添加/编辑 AI 服务商 (`dialog-ai-provider-add` / `dialog-ai-provider-edit`)

| 字段 | 类型 | 说明 |
|------|------|------|
| 服务商 | 下拉选择 | OpenAI/Anthropic/DeepSeek/自定义。切换服务商自动更新模型和端点默认值。 |
| API Key | 密码输入 | 必填，`type=password` |
| 模型 | 下拉选择 | 根据服务商动态切换可选项 |
| API 端点 | 文本输入 | 默认值随服务商变化。自定义模式需手动填写。 |
| 温度 | 数字输入 | 0-2，步长 0.1 |
| 最大 Token 数 | 数字输入 | 256-128000 |
| 备注 | 多行文本 | 可选 |

特殊按钮："测试连接"——点击后发送测试请求到该服务商 API，显示连接结果。

### 16.10 编辑模型分配策略 (`dialog-ai-strategy-edit`)

5 个任务类型，每个包含：
- 模型选择（下拉，从已配置的服务商获取）
- 温度参数（数字输入，0-2）

任务类型：需求文档分析、测试用例生成、自动化脚本生成、知识图谱关联、测试报告摘要。

### 16.11 编辑全局参数 (`dialog-ai-global-edit`)

| 参数 | 类型 | 范围 | 默认值 |
|------|------|------|--------|
| 最大 Token 数 | 数字 | 256-128000 | 4096 |
| 超时时间(秒) | 数字 | 10-300 | 60 |
| 重试次数 | 数字 | 0-10 | 3 |
| 请求并发数 | 数字 | 1-20 | 5 |
| 日志级别 | 下拉 | DEBUG/INFO/WARN/ERROR | INFO |

### 16.12 全局搜索 (`dialog-search`)

- 输入框实时搜索，输入时过滤匹配结果
- 默认显示"最近访问"列表（项目总览、测试用例、知识库、AI 配置）
- 搜索范围：项目名称、用例编号和标题、文档名称、用户姓名
- 点击结果项关闭对话框并跳转到对应页面
