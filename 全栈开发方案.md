# TestFlow 全栈开发方案 — 基于功能描述的前后端设计

## Context

原型文件夹 `Prototype/TestFlow/功能描述.md` 定义了完整的交互功能规格。当前后端数据模型为 `KnowledgeBase → Folder → Document` 三层结构，前端页面已按新模型重构（使用 Mock 数据）。

### 新设计核心变化（2026-06-04 更新）

**Module 从"层级容器"变为"AI 标签字典"**：
- 旧设计：`Project → Sprint → Module(容器) → Document`
- 新设计：`Project → Sprint → Document`（Module 作为 AI 自动识别的标签附着在文档和功能点上）
- 一个文档可跨多个 Module（如"用户认证 + 安全"），AI 自动标注
- `knowledge-docs` 页面已废弃，导航从 4 层简化为 3 层

**当前后端已有的模块**（无需改动）：Auth、User、Role、Project、Activity、Dashboard
**需要重构的模块**：Knowledge（→ Sprint + Module(字典) + Document）
**需要新建的模块**：AI Config、Knowledge Graph、AI Pipeline、Global Search
**需要补充字段的模块**：TestCase（+3字段）、Report（+4字段）

## Phase 1: Sprint/Module/Document 模型重构 ✅ 已完成

**目标**：用 Sprint 替换 KnowledgeBase，Module 变为项目级 AI 标签字典，Document 直接挂在 Sprint 下（module_ids JSON 存储标签关联）。

### 新模型定义

**Sprint** (`app/models/sprint.py` — 新建)
```
sprints
├── id: Integer, PK
├── name: String(100), NOT NULL
├── description: Text
├── project_id: Integer, FK → projects.id
├── status: String(20), 默认"待启动"  # 基线/已完成/进行中/待启动/最新汇总
├── is_all: Boolean, 默认 False       # sprint_all 汇总行
├── created_at: DateTime
└── updated_at: DateTime
```

**Module** (`app/models/module.py` — 新建，项目级 AI 标签字典)
```
modules                              # 非层级容器，是 AI 自动识别的标签字典
├── id: Integer, PK
├── name: String(100), NOT NULL       # 如"用户认证"、"购物车"、"安全"
├── project_id: Integer, FK → projects.id  # 项目级别，不隶属某个 Sprint
├── color: String(20)                 # 前端标签颜色，如 "#2563eb"
└── created_at: DateTime
```

**Document** (`app/models/document.py` — 新建，替代 knowledge.py 中的 Document)
```
documents
├── id: Integer, PK
├── name: String(200), NOT NULL
├── file_path: String(500)
├── file_type: String(20)              # PDF/Word/Markdown/Excel
├── file_size: Integer
├── sprint_id: Integer, FK → sprints.id  # ← 直接挂在 Sprint 下（非 module_id）
├── uploader_id: Integer, FK → users.id
├── version: String(20), 默认 "v1.0"
├── content_preview: Text
├── ai_summary: Text
├── keywords: JSON
├── module_ids: JSON                   # AI 识别的模块 ID 列表，如 [1,3,5]
├── ai_status: String(20), 默认"待分析" # 已分析/分析中/待分析
└── created_at: DateTime
```

> **设计说明**：一个文档可跨多个 Module（如"用户认证 + 安全"），`module_ids` 存储关联的模块 ID 列表。Module 由 AI 从文档内容中自动识别，也可手动管理。

## Phase 4: AI 配置模块

### 新模型（全部在 `app/models/ai_config.py`）

**AIProvider** — AI 服务商
```
ai_providers
├── id, provider_type(OpenAI/Anthropic/DeepSeek/Custom), name, api_key,
├── model, endpoint_url, temperature(Float), max_tokens(Integer),
├── status(正常/限流/错误), last_call_at, created_at
```

**ModelStrategy** — 模型分配策略
```
model_strategies
├── id, task_type(unique, 5种), provider_id(FK), model_name, temperature(Float)
```

**AIGlobalConfig** — 全局参数
```
ai_global_configs
├── id, key(unique), value
```

**AICallLog** — 调用日志
```
ai_call_logs
├── id, task_type, model, provider_id(FK), input_tokens, output_tokens,
├── duration_ms, status(成功/失败/超时), created_at
```

### API 端点（`/api/v1/ai`）
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/providers` | 服务商列表 |
| POST | `/providers` | 新建服务商 |
| PUT | `/providers/{id}` | 编辑服务商 |
| DELETE | `/providers/{id}` | 删除服务商 |
| POST | `/providers/{id}/test` | 测试连接 |
| GET | `/strategies` | 策略列表 |
| PUT | `/strategies` | 批量更新策略 |
| GET | `/config` | 全局参数 |
| PUT | `/config` | 更新全局参数 |
| GET | `/stats` | 统计 |
| GET | `/call-logs` | 调用日志 |

### 种子数据
- AIGlobalConfig: max_tokens=4096, timeout=60, retries=3, concurrency=5, log_level=INFO
- ModelStrategy: 5 种任务类型各一行
